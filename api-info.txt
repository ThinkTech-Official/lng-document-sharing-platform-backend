LNG Document Sharing Platform — API Reference
===============================================

Base URL : http://localhost:3000
Auth     : HttpOnly cookie named lng_token
           Set automatically by POST /auth/login
           Cleared automatically by POST /auth/logout
           withCredentials: true required on all frontend requests
           (except public endpoints which require no cookie)

Role hierarchy  : SUPERADMIN > ADMIN > CONTRACTOR
                  SUPERADMIN passes every role check automatically.
                  Role labels below show the minimum role required.


════════════════════════════════════════════════════════════════
  AUTH  —  /auth
════════════════════════════════════════════════════════════════

POST /auth/login
  Validates email and password, returns a signed JWT on success.
  Generic error "Invalid email or password" for every failure
  — email existence is never revealed.
  Body   : { email: string, password: string }
  Access : Public

POST /auth/forgot-password
  Generates a single-use password reset token (15-min TTL) and
  sends a reset link to the address if it belongs to an active account.
  Always returns success regardless of whether the email exists.
  Body   : { email: string }
  Access : Public

POST /auth/reset-password
  Validates the reset token, enforces password strength rules, hashes
  the new password, marks the token used, and clears force_password_reset.
  Password rules: min 8 chars, one uppercase, one digit, one special char.
  Token is invalidated immediately on first use; expired tokens are rejected.
  Body   : { token: string, new_password: string, confirm_password: string }
  Access : Public

PATCH /auth/change-password
  Changes password for the currently authenticated user.
  Used for the force_password_reset flow on first login.
  Validates password strength and that both fields match,
  then hashes and saves the new password and sets force_password_reset = false.
  Body   : { new_password: string (min 8), confirm_password: string }
  Access : Superadmin, Admin, Contractor (JWT required)

POST /auth/logout
  Stateless logout — records the event in the audit log and returns success.
  Body   : none
  Access : Any authenticated user (JWT required)


════════════════════════════════════════════════════════════════
  ADMINS  —  /admin
════════════════════════════════════════════════════════════════

POST /admin
  Creates a new admin account. Auto-generates a secure temporary password,
  hashes it, sets force_password_reset = true, and emails the credentials.
  The temporary password is never returned in the response.
  Body   : { name: string (min 2), email: string }
  Access : Superadmin only

GET /admin
  Returns a paginated list of non-deleted admin accounts ordered by creation date (desc).
  Query  : { page? (default 1), limit? (default 20, max 100), search? (name or email, case-insensitive) }
  Returns: PaginatedResponse<Admin>  — { data: Admin[], meta: { total, page, limit, totalPages, hasNextPage, hasPrevPage } }
  Access : Superadmin only

GET /admin/:id
  Returns a single admin by ID. 404 if not found or soft-deleted.
  Access : Superadmin only

PATCH /admin/:id
  Updates an admin's name and/or email. Rejects duplicate emails.
  Body   : { name?: string, email?: string }
  Access : Superadmin only

PATCH /admin/:id/status
  Activates or deactivates an admin account. Has no effect on contractors.
  Body   : { is_active: boolean }
  Access : Superadmin only

DELETE /admin/:id
  Soft-deletes an admin (sets deleted_at, sets is_active = false).
  Access : Superadmin only


════════════════════════════════════════════════════════════════
  CONTRACTORS  —  /contractors
════════════════════════════════════════════════════════════════

POST /contractors
  Creates a new contractor account. Auto-generates a temporary password,
  sets force_password_reset = true, and emails the credentials.
  Optionally assigns the contractor to one or more departments on creation.
  Body   : { name: string (min 2), email: string, department_ids?: string[] }
  Access : Admin, Superadmin

GET /contractors
  Returns a paginated list of non-deleted contractors with their department assignments.
  Query  : { page? (default 1), limit? (default 20, max 100), search? (name or email),
             is_active? (true|false), department_id? (filter by department UUID) }
  Returns: PaginatedResponse<Contractor>
  Access : Admin, Superadmin

GET /contractors/:id
  Returns a single contractor including department assignments.
  404 if not found or soft-deleted.
  Access : Admin, Superadmin

PATCH /contractors/:id
  Updates a contractor's name and/or email. Rejects duplicate emails.
  Body   : { name?: string, email?: string }
  Access : Admin, Superadmin

PATCH /contractors/:id/status
  Activates or deactivates a contractor. Has no effect on admins.
  Body   : { is_active: boolean }
  Access : Admin, Superadmin

PATCH /contractors/:id/departments
  Atomically replaces all department assignments for a contractor.
  Validates that all supplied department IDs exist and are not soft-deleted.
  Body   : { department_ids: string[] }
  Access : Admin, Superadmin

DELETE /contractors/:id
  Soft-deletes a contractor (sets deleted_at, sets is_active = false).
  Access : Admin, Superadmin


════════════════════════════════════════════════════════════════
  DEPARTMENTS  —  /departments
════════════════════════════════════════════════════════════════

POST /departments
  Creates a new department. Name must be unique.
  Body   : { name: string, description?: string }
  Access : Admin, Superadmin

GET /departments
  Returns a paginated list of non-deleted departments ordered by name ascending.
  Query  : { page? (default 1), limit? (default 20, max 100), search? (name, case-insensitive) }
  Returns: PaginatedResponse<Department>
  Access : Admin, Superadmin

GET /departments/:id
  Returns a single department by ID. 404 if not found or soft-deleted.
  Access : Admin, Superadmin

PATCH /departments/:id
  Updates a department's name and/or description.
  Rejects duplicate names.
  Body   : { name?: string, description?: string }
  Access : Admin, Superadmin

DELETE /departments/:id
  Soft-deletes a department. Blocked (409 Conflict) if the department has
  any active contractors, any non-deleted restricted documents, or any
  non-deleted restricted videos. All conflicts are returned at once.
  This guard cannot be bypassed even by superadmin.
  Access : Admin, Superadmin


════════════════════════════════════════════════════════════════
  CATEGORIES  —  /categories
════════════════════════════════════════════════════════════════

GET /categories/public
  Returns all root categories with their active subcategories and child subcategories.
  Intended for the contractor-facing homepage. Ordered by sort_order asc.
  (Declared before /:id to prevent routing conflict.)
  Access : Admin, Superadmin, Contractor

POST /categories
  Creates a root category, a subcategory, or a child subcategory (max depth = 2).
  Set parent_category_id to create a subcategory under a root, or a child subcategory
  under a subcategory. 3 levels supported: root → subcategory → child subcategory.
  Child subcategories cannot themselves be parents (max depth = 2).
  Body   : { name: string (min 2), parent_category_id?: string, sort_order?: number (default 0) }
  Access : Admin, Superadmin

GET /categories
  Returns a paginated list of root categories with their full 3-level tree inline.
  Pagination applies to root categories only; subcategories and child subcategories
  are always included in full.
  Search matches root name, subcategory name, OR child subcategory name
  (always returns the root category as the top-level result).
  Query  : { page? (default 1), limit? (default 20, max 100), search? }
  Returns: PaginatedResponse<Category>  — each root item includes subcategories[],
           each subcategory includes its own subcategories[] (child subcategories).
  Ordered by sort_order asc.
  Access : Admin, Superadmin

GET /categories/:id
  Returns a single category, subcategory, or child subcategory by ID.
  Access : Admin, Superadmin

PATCH /categories/:id
  Updates a category's name and/or sort_order.
  Body   : { name?: string, sort_order?: number }
  Access : Admin, Superadmin

DELETE /categories/:id
  Soft-deletes a category. If it is a root category, all of its active subcategories
  AND their child subcategories are also soft-deleted in the same transaction.
  If it is a subcategory, its child subcategories are also soft-deleted.
  Deleting a child subcategory does not affect its parent.
  Documents and videos whose category is soft-deleted return category: null.
  Access : Admin, Superadmin


════════════════════════════════════════════════════════════════
  DOCUMENTS  —  /documents
════════════════════════════════════════════════════════════════

POST /documents
  Creates a document. Accepts either a file upload OR an external URL — not both.
  Exactly one of file or external_url must be present; providing both or neither returns 400.

  FILE upload: Accepted MIME types: PDF, DOC, DOCX, XLS, XLSX, JPG, PNG. Max size: 50 MB.
    file_type is set automatically from the MIME type.
    File is stored in Azure Blob Storage.
  EXTERNAL URL: Provide external_url (must be a valid URL) with no file attached.
    file_type is set to LINK. No Azure upload occurs.
    document_url on GET /documents/:id will be the external URL directly.

  Document starts in DRAFT state. For RESTRICTED access, department_ids is required.
  Content-Type : multipart/form-data
  Fields       : file (required if no external_url),
                 external_url (required if no file, must be a valid URL),
                 title (required, min 2), description?, category_id (required),
                 access_type? (ALL | RESTRICTED, default ALL),
                 department_ids? (string[], required when access_type=RESTRICTED)
  Access : Admin, Superadmin

GET /documents
  Returns a paginated list of documents. Admins see all states; contractors see only PUBLISHED
  documents accessible to their department assignments.
  RESTRICTED documents are filtered at the database query level.
  Query  : { state? (DRAFT|PUBLISHED|UNPUBLISHED), category_id?, search?,
             department_access? (ALL|RESTRICTED — admin/superadmin only),
             page? (default 1), limit? (default 20, max 100) }
           state and department_access filters are silently ignored for contractor role.
  Returns: PaginatedResponse<Document>
  Access : Admin, Superadmin, Contractor

GET /documents/:id
  Returns a single document. Contractors are access-checked: document must
  be PUBLISHED and either ALL or matching the contractor's departments.
  Contractor views are logged as DOCUMENT_VIEWED on every call.
  document_url field: for file documents this is a 60-minute SAS URL;
  for LINK type documents this is the stored external URL — open directly in a new tab.
  Access : Admin, Superadmin, Contractor

PATCH /documents/:id
  Updates a document's title, description, category, and/or external URL.
  Does not change state or access settings.
  external_url may only be supplied for LINK type documents; returns 400 otherwise.
  Body   : { title?: string, description?: string, category_id?: string,
             external_url?: string (LINK type only) }
  Access : Admin, Superadmin

PATCH /documents/:id/reupload
  Replaces the stored file with a new upload. Updates file_url and file_type.
  Does not change the document state or department access.
  Old blob remains in Azure Storage (no cleanup).
  Not available for LINK type documents — returns 400 if attempted.
  Use PATCH /documents/:id with external_url to update a link document's URL.
  Content-Type : multipart/form-data
  Fields       : file (required)
  Access : Admin, Superadmin

PATCH /documents/:id/status
  Transitions the document state. Valid transitions only:
    DRAFT → PUBLISHED
    PUBLISHED → UNPUBLISHED
    UNPUBLISHED → PUBLISHED
  All other transitions return 400.
  Body   : { state: "DRAFT" | "PUBLISHED" | "UNPUBLISHED" }
  Access : Admin, Superadmin

PATCH /documents/:id/departments
  Atomically replaces the access_type and department list.
  Switching to ALL deletes all rows in document_departments for the document.
  Switching to RESTRICTED requires a non-empty department_ids array;
  all IDs are validated and soft-deleted departments are rejected.
  Body   : { access_type: "ALL" | "RESTRICTED", department_ids?: string[] }
  Access : Admin, Superadmin

DELETE /documents/:id
  Soft-deletes a document (sets deleted_at). Blob is not deleted from Azure.
  Access : Admin, Superadmin


════════════════════════════════════════════════════════════════
  VIDEOS  —  /videos
════════════════════════════════════════════════════════════════

POST /videos
  Uploads a new video and thumbnail to Azure Blob Storage.
  Both files are required — the request is rejected if either is missing.
  Accepted video types: MP4 (video/mp4), MOV (video/quicktime),
  AVI (video/x-msvideo). Max size: 2 GB.
  Accepted thumbnail types: JPG, PNG.
  Video is created with upload_status = UPLOADING, transitions to READY
  on success or FAILED on error. blob_url is never returned in any response.
  Content-Type : multipart/form-data
  Fields       : video (required), thumbnail (required), title, description?,
                 category_id, access_type? (ALL | RESTRICTED), department_ids?
  Access : Admin, Superadmin

GET /videos
  Returns a paginated list of videos. Admins see all is_live states; contractors see only videos
  where upload_status = READY and is_live = true, filtered by department access.
  RESTRICTED videos are filtered at the database query level.
  Query  : { is_live? (boolean), category_id?, search?,
             upload_status? (UPLOADING|READY|FAILED — admin/superadmin only),
             department_access? (ALL|RESTRICTED — admin/superadmin only),
             page? (default 1), limit? (default 20, max 100) }
           is_live, upload_status, and department_access filters are silently ignored for contractor role.
  Returns: PaginatedResponse<Video>
  Access : Admin, Superadmin, Contractor

GET /videos/:id/stream
  Returns a short-lived SAS URL (60-minute TTL, Content-Disposition: inline)
  for streaming the video directly from Azure Blob Storage.
  A new SAS URL is generated on every call — nothing is cached or stored.
  blob_url is never exposed; only the time-limited SAS URL is returned.
  Contractor stream requests are logged as VIDEO_STREAMED on every call.
  (Declared before /:id to prevent routing conflict.)
  Access : Admin, Superadmin, Contractor

GET /videos/:id
  Returns video metadata. blob_url is stripped from the response.
  Contractors are access-checked: video must be READY, is_live = true,
  and either ALL access or matching the contractor's department assignments.
  Access : Admin, Superadmin, Contractor

PATCH /videos/:id
  Updates a video's title, description, and/or category.
  Does not change upload_status, is_live, or access settings.
  Body   : { title?: string, description?: string, category_id?: string }
  Access : Admin, Superadmin

PATCH /videos/:id/status
  Sets is_live to true or false. Rejected with 400 if upload_status is not READY.
  upload_status itself cannot be set via any endpoint.
  Body   : { is_live: boolean }
  Access : Admin, Superadmin

PATCH /videos/:id/departments
  Atomically replaces the access_type and department list.
  Switching to ALL deletes all rows in video_departments for the video.
  Switching to RESTRICTED requires a non-empty department_ids array;
  all IDs are validated and soft-deleted departments are rejected.
  Body   : { access_type: "ALL" | "RESTRICTED", department_ids?: string[] }
  Access : Admin, Superadmin

DELETE /videos/:id
  Soft-deletes a video (sets deleted_at). Blob and thumbnail are not deleted.
  Access : Admin, Superadmin


════════════════════════════════════════════════════════════════
  NOTIFICATIONS  —  /notifications
════════════════════════════════════════════════════════════════

GET /notifications
  Returns a paginated list of notifications
  ordered by created_at descending (newest first).
  Contractors receive all notifications (no pagination
  applied unless params are passed).
  Admins can search and paginate.
  Query  : {
    page?   : number (default 1)
    limit?  : number (default 20, max 100)
    search? : string (searches title and content)
  }
  Returns: PaginatedResponse<Notification>
  Access : Admin, Superadmin, Contractor

POST /notifications
  Creates a new notification broadcast.
  Body   : {
    title    : string (max 200 chars)
    content  : string (HTML from rich text editor)
    category : 'red' | 'orange' | 'yellow' |
               'green' | 'blue' | 'black'
  }
  Returns: Notification
  Access : Admin, Superadmin only

GET /notifications/:id
  Returns a single notification by ID.
  404 if not found.
  Returns: Notification
  Access : Admin, Superadmin only

PATCH /notifications/:id
  Updates a notification's title, content, and/or category.
  All fields optional. 404 if not found.
  Body   : {
    title?   : string (max 200 chars)
    content? : string (HTML)
    category?: 'red'|'orange'|'yellow'|'green'|'blue'|'black'
  }
  Returns: Updated Notification
  Access : Admin, Superadmin only

DELETE /notifications/:id
  Hard-deletes a notification.
  404 if not found.
  Returns: { message: 'Notification deleted' }
  Access : Admin, Superadmin only

POST /notifications/upload-image
  Uploads an image file to Azure Blob Storage
  for use inside notification rich text content.
  Returns a permanent URL to embed in the HTML.
  Content-Type : multipart/form-data
  Fields       : image (required — JPG, PNG, GIF, WebP, max 5 MB)
  Returns      : { url: string }
  Access       : Admin, Superadmin only

Notification shape:
{
  id:         string
  title:      string
  content:    string (HTML)
  category:   string
  created_at: string (ISO datetime)
}


════════════════════════════════════════════════════════════════
  AUDIT LOGS  —  /logs
════════════════════════════════════════════════════════════════

GET /logs
  Returns a paginated list of activity log entries with optional filters.
  Cursor-based pagination: pass the created_at ISO string of the last item
  as cursor to get the next page. Date range and cursor filters are merged
  (not clobbered) so they can be combined safely.
  Query  : {
             actor_role?   : SUPERADMIN | ADMIN | CONTRACTOR
             action_type?  : (see ActionType enum below)
             target_type?  : string
             actor_id?     : string
             date_from?    : ISO 8601 datetime
             date_to?      : ISO 8601 datetime
             cursor?       : ISO 8601 datetime  (created_at of last seen item)
             limit?        : integer 1–100 (default 50)
           }
  Returns: { data: ActivityLog[], nextCursor: string | null, hasNextPage: boolean }
           ActivityLog shape: { id, actor_id, actor_role, actor_name, actor_email,
                                action_type, target_type, target_id,
                                ip_address, user_agent, metadata, created_at }
  Access : Superadmin only

GET /logs/:id
  Returns a single activity log entry by ID. 404 if not found.
  Access : Superadmin only

════════════════════════════════════════════════════════════════
  PAGINATION
════════════════════════════════════════════════════════════════

All list endpoints (except GET /categories/public and GET /logs) return a
paginated response envelope:

{
  "data": [...],
  "meta": {
    "total": 100,
    "page": 1,
    "limit": 20,
    "totalPages": 5,
    "hasNextPage": true,
    "hasPrevPage": false
  }
}

  page   : 1-based page number (default 1)
  limit  : items per page (default 20, max 100)

GET /categories/public — returns a flat array (full tree, no pagination)
GET /logs              — uses cursor-based pagination (see Audit Logs section)


════════════════════════════════════════════════════════════════
  GLOBAL RULES AND NOTES
════════════════════════════════════════════════════════════════

Authentication
  All protected routes require the HttpOnly cookie lng_token.
  The cookie is set automatically by POST /auth/login and cleared by POST /auth/logout.
  The browser sends it automatically — no Authorization header is needed or used.
  withCredentials: true is required on all frontend requests.
  JWT payload contains: { sub (user_id), role, force_password_reset }
  JWT expiry: 24 hours. Secret from JWT_SECRET environment variable.
  Soft-deleted users are rejected in the JWT strategy validate() step.
  Inactive users (is_active = false) are rejected at the same step.

force_password_reset
  When an admin or contractor is created, force_password_reset = true.
  The JwtAuthGuard throws 403 on every protected route for users with
  force_password_reset = true. The only escape is POST /auth/reset-password
  (public, no JWT required). On successful reset, the flag is set to false.
  Superadmin is seeded with force_password_reset = false.

Role Hierarchy
  SUPERADMIN: passes every @Roles() check automatically (no bypass needed).
  ADMIN: accesses all admin-labelled endpoints plus contractor-shared ones.
  CONTRACTOR: accesses only explicitly opened read endpoints.
  RolesGuard uses getAllAndOverride so method-level @Roles() overrides
  class-level @Roles() on the same controller.

Soft Delete
  User, Department, Category, Document, and Video all use a deleted_at
  DateTime? column. Records with a non-null deleted_at are excluded from
  every findAll and findOne query throughout the application.
  Hard deletes are never performed on application data.

Department Deletion Guard
  DELETE /departments/:id checks three counts in parallel:
    • Active contractors assigned to the department
    • Non-deleted restricted documents linked to the department
    • Non-deleted restricted videos linked to the department
  All conflicts are returned simultaneously in a single 409 response.
  The guard runs at the service layer — it cannot be bypassed by any role,
  including superadmin.

Soft-Deleted Department Assignment
  All endpoints that accept department_ids (contractor create/update,
  document create/updateDepartments, video create/updateDepartments)
  validate that every supplied ID exists AND has deleted_at = null.
  Assigning a soft-deleted department returns 400.

Category Depth Limit
  Two levels of hierarchy are supported: root → subcategory → child subcategory.
  Attempting to create a 4th level (child subcategory as a parent) returns 400.

Logging — Fire-and-Forget
  LoggingService.log() is synchronous from the caller's perspective.
  It calls the async write internally and attaches .catch() to suppress
  errors. A logging failure never crashes or delays the calling request.
  LoggingModule is @Global() — no module needs to import it explicitly.

IP Address and User-Agent Capture
  RequestMetaInterceptor runs globally on every request.
  It reads the x-forwarded-for header (first IP in the list) and falls
  back to req.ip for direct connections. The result is stored as
  req.requestMeta = { ip_address, user_agent }. All controllers pass
  req.requestMeta directly to service log calls, which spread it into
  LogActionParams. ip_address and user_agent are persisted on every
  activity log row.

Log Archiving
  A @Cron('0 2 * * *') job runs at 02:00 daily.
  It fetches all activity_logs older than 30 days, gzip-compresses them
  as JSON, uploads the archive to the Azure Blob "logs-archive" container,
  and only then deletes the rows from the database.
  If the upload fails, the rows are retained for the next day's retry.

Azure Blob — Lazy Initialisation
  AzureBlobService, AzureVideoService, and LoggingArchiveService all use
  a lazy getter for BlobServiceClient. The client is not constructed until
  the first actual file operation, so the application starts successfully
  even when AZURE_STORAGE_CONNECTION_STRING holds a placeholder value.

Video blob_url
  blob_url is fetched internally for SAS generation but is stripped in
  every toResponse() call and never appears in any API response.
  Clients receive only the time-limited SAS URL from GET /videos/:id/stream.
